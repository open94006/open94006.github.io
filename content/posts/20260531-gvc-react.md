# GVC → React 對照手冊

> 本文件對應 `PLAYGROUND-GVC.md` 的每一個 GVC 用法，示範在 React 中的等效寫法。
> 每個章節皆為「GVC 寫法 → React 等效寫法」的雙欄對照。

---

## 速查對照表

| GVC / Glitter                      | React 對應                                        |
| ---------------------------------- | ------------------------------------------------- |
| `gvc.bindView({ bind, view })`     | Functional Component + JSX `return`               |
| `let vm = {}` + `notifyDataChange` | `useState`                                        |
| `dataList: [{ obj, key }]`         | `useState` (setter 自動觸發重繪)                  |
| `onInitial`                        | `useEffect(() => {}, [])`                         |
| `onCreate`                         | `useEffect(() => {})` (無 deps)                   |
| `onDestroy`                        | `useEffect(() => { return () => cleanup() }, [])` |
| `gvc.event(() => fn())`            | JSX `onClick={() => fn()}`                        |
| `gvc.glitter.share.xxx`            | `useContext` / Zustand store                      |
| `gvc.glitter.getUUID()`            | `useId()` / `useRef`                              |
| `gvc.glitter.innerDialog(...)`     | `useState(isOpen)` + Portal / Modal 元件          |
| `gvc.addStyle(...)`                | CSS Module / `styled-components` / `<style>` 標籤 |
| `glitter.runJsInterFace(...)`      | React Native `NativeModules` / Capacitor Plugin   |

## 1. 元件宣告與局部重繪

### GVC

```typescript
function CounterComponent(gvc: any) {
    const vm = { count: 0 };
    const bindId = gvc.glitter.getUUID();

    return gvc.bindView({
        bind: bindId,
        view: () => `
      <div class="counter-container">
        <h3>目前的計數：${vm.count}</h3>
        <button class="add-btn">+1</button>
      </div>
    `,
        onCreate: () => {
            gvc.getBindViewElem(bindId)
                .find('.add-btn')
                .off('click')
                .on('click', () => {
                    vm.count++;
                    gvc.notifyDataChange(bindId);
                });
        },
    });
}
```

### React 對應

```tsx
import { useState } from 'react';

function CounterComponent() {
    const [count, setCount] = useState(0);

    return (
        <div className="counter-container">
            <h3>目前的計數：{count}</h3>
            <button onClick={() => setCount(count + 1)}>+1</button>
        </div>
    );
}
```

> **差異說明**：GVC 手動呼叫 `notifyDataChange` 觸發特定區塊重繪；React 呼叫 `setCount` 後框架自動 Diff Virtual DOM 並更新畫面。

## 2. 生命週期對照

### GVC

```typescript
gvc.bindView({
    bind: 'my-view',
    view: () => `<div>...</div>`,

    onInitial: () => {
        // 僅第一次掛載後執行一次
        fetchInitialData();
    },

    onCreate: () => {
        // 每次重繪後執行（負責重新綁定 DOM 事件）
        rebindEvents();
    },

    onDestroy: () => {
        // 元件銷毀時清理資源
        clearInterval(timer);
        socket.disconnect();
    },
});
```

### React 對應

```tsx
import { useEffect, useRef } from 'react';

function MyComponent() {
    const timerRef = useRef<ReturnType<typeof setInterval>>();

    // ① onInitial 對應：空 deps 陣列，只執行一次
    useEffect(() => {
        fetchInitialData();
    }, []);

    // ② onCreate 對應：無 deps，每次 render 後執行
    useEffect(() => {
        rebindEvents();
    });

    // ③ onDestroy 對應：useEffect 的 cleanup return
    useEffect(() => {
        return () => {
            clearInterval(timerRef.current);
            socket.disconnect();
        };
    }, []);

    return <div>...</div>;
}
```

## 3. 自動響應資料 (`dataList` vs `useState`)

### GVC

```typescript
const vm = { name: 'Hello' };
const bindId = gvc.glitter.getUUID();

return gvc.bindView({
    bind: bindId,
    view: () => `
    <div>
      <p>${vm.name}</p>
      <button class="rename-btn">改名</button>
    </div>
  `,
    dataList: [{ obj: vm, key: 'name' }], // observe vm.name，賦值後自動重繪
    onCreate: () => {
        gvc.getBindViewElem(bindId)
            .find('.rename-btn')
            .off('click')
            .on('click', () => {
                vm.name = 'World'; // 直接賦值即可，dataList 會自動觸發重繪
            });
    },
});
```

### React 對應

```tsx
import { useState } from 'react';

function NameDisplay() {
    const [name, setName] = useState('Hello');

    return (
        <div>
            <p>{name}</p>
            {/* setName 呼叫後 React 自動重繪，對應 dataList 的 observer 機制 */}
            <button onClick={() => setName('World')}>改名</button>
        </div>
    );
}
```

## 4. 行內事件綁定 (`gvc.event` vs JSX 事件屬性)

### GVC

```typescript
// gvc.event 把閉包封裝成字串 ID，讓純字串模板能掛載 JS 函數
view: () => `
  <!-- onclick -->
  <button onclick="${gvc.event(() => {
      console.log('送出');
      submitForm();
  })}">送出</button>

  <!-- onchange -->
  <input
    type="text"
    onchange="${gvc.event((e: any) => {
        vm.keyword = e.target.value;
        gvc.notifyDataChange(bindId);
    })}"
  />

  <!-- 動態列表：每個 item 各自的 onclick -->
  ${items
      .map(
          (item, i) => `
    <li>
      ${item.name}
      <button onclick="${gvc.event(() => {
          items.splice(i, 1);
          gvc.notifyDataChange(bindId);
      })}">刪除</button>
    </li>
  `,
      )
      .join('')}
`;
```

### React 對應

```tsx
// React JSX 直接在屬性寫箭頭函數，不需要 gvc.event 的轉換層
function MyForm() {
    const [keyword, setKeyword] = useState('');
    const [items, setItems] = useState(['蘋果', '香蕉', '芒果']);

    return (
        <div>
            {/* onclick */}
            <button onClick={() => submitForm()}>送出</button>

            {/* onchange */}
            <input type="text" onChange={(e) => setKeyword(e.target.value)} />

            {/* 動態列表 */}
            <ul>
                {items.map((item, i) => (
                    <li key={i}>
                        {item}
                        <button onClick={() => setItems(items.filter((_, idx) => idx !== i))}>刪除</button>
                    </li>
                ))}
            </ul>
        </div>
    );
}
```

## 5. 全域共用狀態 (`glitter.share` vs Context / Zustand)

### GVC

```typescript
// 寫入：任意地方皆可，無需 import
gvc.glitter.share.currentUser = { name: 'Daniel' };

// 把重繪函數掛到 share，讓子元件可以觸發父元件更新
gvc.glitter.share.refreshList = () => {
    gvc.notifyDataChange(listBindId);
};

// 讀取與呼叫：任意元件直接存取
const user = gvc.glitter.share.currentUser;
gvc.glitter.share.refreshList?.();
```

### React 對應 (Zustand)

```tsx
// store.ts：定義全域 store
import { create } from 'zustand';

interface AppStore {
    currentUser: { name: string } | null;
    setCurrentUser: (user: { name: string }) => void;
    onRefreshList: (() => void) | null;
    setRefreshList: (fn: () => void) => void;
}

const useAppStore = create<AppStore>((set) => ({
    currentUser: null,
    setCurrentUser: (user) => set({ currentUser: user }),
    onRefreshList: null,
    setRefreshList: (fn) => set({ onRefreshList: fn }),
}));

// 父元件：把自己的 setter 存入 store（對應 share.refreshList = ...）
function ParentList() {
    const [list, setList] = useState<string[]>([]);
    const setRefreshList = useAppStore((s) => s.setRefreshList);

    useEffect(() => {
        setRefreshList(() => {
            fetchList().then(setList);
        });
    }, []);

    return (
        <ul>
            {list.map((item, i) => (
                <li key={i}>{item}</li>
            ))}
        </ul>
    );
}

// 子元件：呼叫 store 裡的函數觸發父元件更新
function ChildForm() {
    const refreshList = useAppStore((s) => s.onRefreshList);

    const handleSubmit = async () => {
        await postNewItem();
        refreshList?.(); // 對應 gvc.glitter.share.refreshList?.()
    };

    return <button onClick={handleSubmit}>新增</button>;
}
```

## 6. Modal / Dialog 控制

### GVC

```typescript
// 開啟：呼叫 innerDialog 自動疊加覆蓋層
gvc.glitter.innerDialog((dialogGvc: any) => {
    const bindId = dialogGvc.glitter.getUUID();

    return dialogGvc.bindView({
        bind: bindId,
        view: () => `
      <div style="padding:24px;">
        <h4>編輯名稱</h4>
        <input class="name-input" type="text" value="${vm.name}" />
        <button class="cancel-btn">取消</button>
        <button class="confirm-btn">確認</button>
      </div>
    `,
        onCreate: () => {
            const el = dialogGvc.getBindViewElem(bindId);
            el.find('.cancel-btn')
                .off('click')
                .on('click', () => {
                    gvc.glitter.closeDiaLog('edit-dialog');
                });
            el.find('.confirm-btn')
                .off('click')
                .on('click', async () => {
                    await saveItem();
                    gvc.glitter.closeDiaLog('edit-dialog');
                    gvc.glitter.share.refreshList?.();
                });
        },
    });
}, 'edit-dialog');
```

### React 對應

```tsx
import { useState } from 'react';
import { createPortal } from 'react-dom';

function EditDialog({ name, onClose, onSave }: { name: string; onClose: () => void; onSave: (name: string) => void }) {
    const [editedName, setEditedName] = useState(name);

    const handleConfirm = async () => {
        await saveItem(editedName);
        onSave(editedName);
        onClose();
    };

    // createPortal 讓 Dialog 渲染在 body 最外層（對應 innerDialog 的覆蓋效果）
    return createPortal(
        <div className="modal-overlay">
            <div style={{ padding: 24 }}>
                <h4>編輯名稱</h4>
                <input value={editedName} onChange={(e) => setEditedName(e.target.value)} />
                <button onClick={onClose}>取消</button>
                <button onClick={handleConfirm}>確認</button>
            </div>
        </div>,
        document.body,
    );
}

// 父元件控制開關
function ParentComponent() {
    const [isOpen, setIsOpen] = useState(false);

    return (
        <div>
            <button onClick={() => setIsOpen(true)}>開啟視窗</button>
            {isOpen && <EditDialog name="原始名稱" onClose={() => setIsOpen(false)} onSave={(newName) => console.log('儲存：', newName)} />}
        </div>
    );
}
```

## 7. 唯一 ID 產生 (`getUUID` vs `useId`)

### GVC

```typescript
// 在元件最頂層產生，確保同一頁面多個實例不衝突
function ProductCard(gvc: any, product: any) {
    const cardId = gvc.glitter.getUUID();
    const btnId = gvc.glitter.getUUID();

    return gvc.bindView({
        bind: cardId,
        view: () => `
      <div>
        <p>${product.name}</p>
        <button id="${btnId}" class="add-cart-btn">加入購物車</button>
      </div>
    `,
        onCreate: () => {
            gvc.getBindViewElem(cardId)
                .find('.add-cart-btn')
                .off('click')
                .on('click', () => {
                    gvc.glitter.share.addToCart?.(product);
                });
        },
    });
}

// 產生多個 card（每個都有獨立 ID）
view: () => `
  <div class="product-list">
    ${productList.map((product) => ProductCard(gvc, product)).join('')}
  </div>
`;
```

### React 對應

```tsx
import { useId } from 'react';

// React 的每個元件實例天然獨立，不需要手動管理 ID
// useId 只在需要明確的 HTML id 屬性（如 <label for>）時才需要
function ProductCard({ product }: { product: { name: string } }) {
    const inputId = useId(); // 用於 label/input 配對等需要 DOM id 的場景

    return (
        <div>
            <label htmlFor={inputId}>數量</label>
            <input id={inputId} type="number" />
            <p>{product.name}</p>
            <button onClick={() => addToCart(product)}>加入購物車</button>
        </div>
    );
}

// 產生多個 card：key prop 負責 React 的 reconciliation，不需要手動 UUID
function ProductList({ products }: { products: any[] }) {
    return (
        <div className="product-list">
            {products.map((product) => (
                <ProductCard key={product.id} product={product} />
            ))}
        </div>
    );
}
```

### React 對應 (styled-components)

```tsx
import styled, { keyframes } from 'styled-components';

const fadeIn = keyframes`
  from { opacity: 0; }
  to   { opacity: 1; }
`;

// 樣式與元件合一，不需要手動注入，框架自動處理 class 命名衝突
const ToastMessage = styled.div`
    position: fixed;
    bottom: 24px;
    right: 24px;
    padding: 12px 20px;
    background: #333;
    color: #fff;
    border-radius: 8px;
    animation: ${fadeIn} 0.3s ease;
`;

function Toast({ message }: { message: string }) {
    return <ToastMessage>{message}</ToastMessage>;
}
```

## 9. 巢狀區塊獨立重繪

### GVC

```typescript
// 頁面拆成多個 bindView，各自維護 ID，notifyDataChange 時精準只刷新需要的區塊
const ids = {
  list:    gvc.glitter.getUUID(),
  summary: gvc.glitter.getUUID(),
};

view: () => `
  <div>
    ${gvc.bindView({
      bind: ids.list,
      view: () => vm.loading ? '<p>載入中...</p>' : renderList(vm.orders),
    })}
    ${gvc.bindView({
      bind: ids.summary,
      view: () => `<strong>合計：NT$ ${vm.total}</strong>`,
    })}
  </div>
`,
onInitial: async () => {
  const data = await fetchOrders();
  vm.orders = data.list;
  vm.total = data.total;
  vm.loading = false;
  // 只刷新這兩個區塊，其餘 DOM 不動
  gvc.notifyDataChange([ids.list, ids.summary]);
},
```

### React 對應

```tsx
import { useState, useEffect } from 'react';

// React 的子元件天然獨立：OrderList 和 OrderSummary 各自管理 state
// 父元件傳 props 給子元件，子元件 re-render 只影響自己的 subtree

function OrderList({ orders }: { orders: any[] }) {
    return (
        <ul>
            {orders.map((o) => (
                <li key={o.id}>
                    {o.name} x{o.qty}
                </li>
            ))}
        </ul>
    );
}

function OrderSummary({ total }: { total: number }) {
    return <strong>合計：NT$ {total}</strong>;
}

function OrderPage() {
    const [orders, setOrders] = useState<any[]>([]);
    const [total, setTotal] = useState(0);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        fetchOrders().then((data) => {
            setOrders(data.list);
            setTotal(data.total);
            setLoading(false);
            // React 的 Virtual DOM Diff 確保只更新有變動的部分
        });
    }, []);

    if (loading) return <p>載入中...</p>;

    return (
        <div>
            <OrderList orders={orders} />
            <OrderSummary total={total} />
        </div>
    );
}
```
